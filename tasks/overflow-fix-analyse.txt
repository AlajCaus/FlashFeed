ANALYSE: 96px Overflow-Fehler in Angebote-Ansicht
===================================================

PROBLEM:
- Bottom Overflow von 96px in OfferComparisonCard Widgets
- Tritt in der Grid-Ansicht der Angebote auf

FEHLERQUELLE:
Datei: lib/widgets/offer_comparison_card.dart
Zeilen: 284-408

Hauptproblem:
- SingleChildScrollView (Zeile 284) innerhalb eines Containers ohne feste Höhenbeschränkung
- Das Grid verwendet childAspectRatio für die Kartengröße, aber der ScrollView expandiert darüber hinaus
- Der ScrollView ist unnötig, da die Card-Inhalte in die vorgegebene Größe passen sollten

LÖSUNG:
1. SingleChildScrollView entfernen
2. Stattdessen direkt das Padding mit Column verwenden
3. Falls Inhalte zu groß werden: Flexible/Expanded für kritische Bereiche verwenden
4. Overflow.ellipsis für Texte ist bereits vorhanden

AUSWIRKUNGSANALYSE:
Betroffene Dateien:
- lib/widgets/offer_comparison_card.dart (Hauptänderung)

Betroffene Komponenten:
- OfferComparisonCard Widget
- offers_screen.dart nutzt diese Cards im Grid
- flash_deals_screen.dart könnte auch betroffen sein (nutzt ähnliche Cards)

Tests:
- Keine direkten Unit-Tests für OfferComparisonCard gefunden
- Visual Tests erforderlich nach Änderung

Breaking Changes:
- Keine, da nur interne Layout-Struktur geändert wird
- API der Komponente bleibt gleich

Abhängigkeiten:
- Keine externen Abhängigkeiten betroffen
- Nur Flutter-Layout-Widgets
